🧠 기억 시스템 — AI가 어떻게 기억하나요?

클로드 코드는 매 세션 새로 시작합니다. 하지만 "기억"을 파일로 남길 수 있어요.

두 가지 기억 방식

📝CLAUDE.md — 내가 직접 쓰는 기억

프로젝트 폴더에 CLAUDE.md 파일을 만들면, 클로드가 매 세션 시작할 때 자동으로 읽습니다.

"우리 프로젝트 규칙", "빌드 방법", "코딩 스타일" 등을 적어두면 매번 설명할 필요 없어요!

🤖Auto Memory — AI가 스스로 쓰는 기억

여러분이 "아니, 그렇게 하지 말고 이렇게 해"라고 교정하면, 클로드가 자동으로 메모해둡니다.

다음 세션에서 같은 실수를 반복하지 않아요! (첫 200줄 자동 로드)

CLAUDE.mdAuto Memory
누가 쓰나내가 직접AI가 자동으로
뭘 저장하나규칙, 표준, 워크플로빌드 명령어, 디버깅 인사이트, 선호도
적용 범위프로젝트/개인/조직현재 작업 트리
언제 불러오나매 세션 시작매 세션 시작 (200줄)

CLAUDE.md 실전 작성법

CLAUDE.md 예시# 프로젝트 규칙 ## 빌드 & 테스트 - 빌드: npm run build - 테스트: npm test - 커밋 전 반드시 테스트 통과 확인 ## 코딩 스타일 - TypeScript 사용 - 들여쓰기: 스페이스 2칸 - 함수명: camelCase - 한국어 주석 사용 ## 프로젝트 구조 - API 핸들러: src/api/handlers/ - DB 모델: src/models/ - 테스트: tests/ ## 주의사항 - .env 파일 절대 커밋하지 말 것 - API 키를 코드에 직접 넣지 말 것

📂 CLAUDE.md 저장 위치별 범위

위치적용 범위공유 대상
./CLAUDE.md이 프로젝트만팀원 전체 (Git)
~/.claude/CLAUDE.md내 모든 프로젝트나만
/etc/claude-code/조직 전체전 직원
💡 꿀팁: /init 명령어
프로젝트 폴더에서 /init 을 실행하면 클로드가 코드를 분석해서 CLAUDE.md를 자동 생성해줘요! 빌드 명령어, 프로젝트 구조, 코딩 규칙을 자동으로 파악합니다.

📁 Rules — 파일별 규칙 분리

프로젝트가 커지면 CLAUDE.md 하나로 부족할 수 있어요. .claude/rules/ 폴더에 주제별로 분리하세요!

폴더 구조프로젝트/ ├── .claude/ │ ├── CLAUDE.md # 메인 규칙 │ └── rules/ │ ├── code-style.md # 코딩 스타일 │ ├── testing.md # 테스트 규칙 │ ├── api-design.md # API 설계 원칙 │ └── frontend/ │ └── react.md # 프론트 전용 규칙
💡 @import 기능
CLAUDE.md에서 @docs/guide.md 처럼 다른 파일을 불러올 수 있어요. README나 package.json도 가져올 수 있습니다!
⚠️ CLAUDE.md는 200줄 이하로! 너무 길면 토큰을 많이 먹고, AI가 규칙을 놓칠 확률이 높아져요.

기본 내장 서브에이전트 3개

🔍Explore (탐색)

모델: Haiku (빠르고 저렴) | 권한: 읽기 전용

코드 검색, 파일 찾기, 구조 분석용. 빠르게 훑어볼 때 자동으로 호출됩니다.

📋Plan (계획)

모델: 메인과 동일 | 권한: 읽기 전용

플랜 모드에서 코드를 분석하고 계획을 세울 때 사용됩니다.

🔧General-purpose (범용)

모델: 메인과 동일 | 권한: 전체

복잡한 멀티스텝 작업. 탐색 + 수정 + 실행을 한꺼번에 처리합니다.

🛠️ 나만의 서브에이전트 만들기

방법 1: 대화형 만들기 (추천)

클로드 코드 내/agents

"Create new agent" → 범위 선택 → Claude가 자동 생성! 도구 접근 권한, 모델, 색상까지 설정 가능.

방법 2: 파일로 직접 만들기

.claude/agents/code-reviewer.md--- name: code-reviewer description: 코드 품질과 보안을 검토합니다 tools: Read, Glob, Grep model: sonnet --- 코드 리뷰어입니다. 호출되면 코드를 분석하고 품질, 보안, 모범 사례에 대한 구체적이고 실행 가능한 피드백을 제공하세요.

방법 3: 명령줄에서 즉석 생성

터미널claude --agents '{ "reviewer": { "description": "코드 리뷰 전문가", "prompt": "시니어 코드 리뷰어입니다. 품질, 보안, 모범 사례에 집중하세요.", "tools": ["Read", "Grep", "Glob", "Bash"], "model": "sonnet" } }'

서브에이전트 호출 방법

직접 부르거나, AI가 알아서 판단해서 부릅니다!

🤖 AI가 자동 판단
"최근 변경사항 보안 이슈 검토해줘"
→ AI가 알아서 리뷰어 서브에이전트 호출
👤 직접 지정
"code-reviewer 에이전트로 인증 모듈 검토해줘"
→ 특정 에이전트 직접 호출

📂 서브에이전트 저장 위치

위치범위용도
.claude/agents/이 프로젝트팀과 공유 (Git)
~/.claude/agents/내 모든 프로젝트개인 전용
--agents 플래그이번 세션만테스트/자동화
💡 비용 절감 팁
탐색용 에이전트는 model: haiku로 설정하면 빠르고 저렴하게! 실제 코드 수정만 비싼 모델 쓰세요.

⚡ 스킬 — 나만의 명령어 만들기

/deploy, /review-pr 같은 커스텀 명령어를 만들 수 있어요!

스킬이 뭔가요?

반복적인 작업을 SKILL.md 파일 하나로 패키지한 것. /스킬이름으로 호출하거나, AI가 상황에 맞게 자동으로 불러옵니다.

내장 스킬 5개

스킬하는 일
/batch대규모 변경을 병렬로 처리 (5~30개 단위로 쪼개서 동시 작업!)
/claude-apiClaude API 레퍼런스 로드
/debug현재 세션 문제 진단
/loop일정 간격으로 반복 실행 (배포 모니터링 등)
/simplify최근 변경 코드 품질 리뷰 + 자동 수정

🛠️ 나만의 스킬 만들기

~/.claude/skills/deploy/SKILL.md--- name: deploy description: 프로덕션 배포 절차를 실행합니다 --- 배포 시 다음 순서를 따르세요: 1. npm test 실행 — 전체 통과 확인 2. npm run build — 빌드 성공 확인 3. git tag로 버전 태그 생성 4. npm run deploy 실행 5. 배포 후 헬스체크 URL 확인 6. 결과 요약 보고

이제 /deploy 라고만 치면 위 순서대로 자동 실행!

스킬 저장 위치

위치범위
.claude/skills/이 프로젝트 (팀 공유)
~/.claude/skills/내 모든 프로젝트

실전 스킬 예시들

🔍/review-pr

PR 변경사항을 자동 리뷰. 보안 이슈, 성능 문제, 코드 스타일 체크.

📊/report

주간 코드 변경 리포트 자동 생성. 커밋 수, 변경 파일, 주요 변경사항 요약.

🌐/translate

다국어 번역 자동화. 새 문자열을 감지해서 번역 파일 업데이트.

📝/changelog

최근 커밋 기반으로 CHANGELOG.md 자동 업데이트.

💡 스킬 vs 규칙(Rules) 차이
Rules = 항상 로드됨 (코딩 스타일처럼 매번 필요한 것)
Skills = 필요할 때만 로드 (배포, 리뷰처럼 가끔 필요한 것)
→ 토큰 절약!

Git으로 공유하는 것들

📝 CLAUDE.md
📁 .claude/rules/
🤖 .claude/agents/

이 파일들을 Git에 커밋하면, 팀원 누구나 git pull 후 동일한 AI 환경을 사용!

팀 프로젝트 셋업 예시

프로젝트 구조우리-프로젝트/ ├── CLAUDE.md # 팀 공통 규칙 ├── .claude/ │ ├── rules/ │ │ ├── code-style.md # 코딩 스타일 │ │ ├── git-workflow.md # Git 워크플로 │ │ └── security.md # 보안 규칙 │ ├── agents/ │ │ ├── reviewer.md # 코드 리뷰어 │ │ └── tester.md # 테스트 작성자 │ └── skills/ │ ├── deploy/SKILL.md # 배포 절차 │ └── review-pr/SKILL.md # PR 리뷰 ├── src/ └── tests/

개인 설정 vs 팀 설정 분리

📁 팀 공유 (Git 커밋)
./CLAUDE.md
.claude/rules/
.claude/agents/
.claude/skills/
🔒 개인만 (Git 무시)
~/.claude/CLAUDE.md
~/.claude/agents/
~/.claude/skills/
Auto Memory
💡 개인 설정을 팀 규칙에 연결하기
팀 CLAUDE.md에 @~/.claude/my-project-rules.md를 넣으면, 개인 설정 파일을 import할 수 있어요. 파일 자체는 Git에 안 올라가니까 안전!

① 검증 수단을 꼭 알려주세요

클로드 코드가 가장 잘 일하는 조건: 스스로 결과를 확인할 수 있을 때!

❌ 검증 없는 요청
"이메일 검증 함수 만들어줘"
✅ 검증 있는 요청
"이메일 검증 함수 만들어줘. user@example.com은 true, invalid는 false로 테스트하고 실행까지 해줘"

② 4단계 워크플로

복잡한 작업은 이 순서로!

🔍 탐색
→
📋 계획
→
💻 구현
→

# 1. 플랜 모드로 탐색 [플랜 모드] src/auth 읽고 세션 관리 방식 이해해줘 # 2. 계획 세우기 [플랜 모드] Google OAuth 추가하려면 어떤 파일을 바꿔야 해? # 3. 실행 [일반 모드] 계획대로 OAuth 구현하고 테스트 작성해서 돌려봐 # 4. 커밋 [일반 모드] 설명 넣어서 커밋하고 PR 올려줘

③ 컨텍스트 관리가 핵심

클로드의 대화 창(컨텍스트 윈도우)은 한정적이에요. 꽉 차면 성능이 떨어집니다!

🧹컨텍스트 절약 팁

• 하나의 세션에서 너무 많은 작업 X → 주제별로 세션 나누기

• /clear로 대화 초기화 후 새 주제 시작

• 탐색은 Haiku 서브에이전트에게 (메인 컨텍스트 절약)

• CLAUDE.md 200줄 이하로 유지

• Rules는 주제별로 분리 (필요한 것만 로드)

④ /batch로 대규모 작업 병렬 처리

터미널/batch src/ 폴더의 모든 React 컴포넌트를 TypeScript로 변환해줘

AI가 작업을 5~30개로 쪼개고, 각각 별도 에이전트가 동시에 처리! PR도 자동 생성.

⑤ 파이프라인으로 자동화

# 로그 실시간 모니터링 tail -f app.log | claude -p "이상 징후 발견하면 알려줘" # 변경 파일 보안 검토 git diff main --name-only | claude -p "보안 이슈 확인" # CI에서 번역 자동화 claude -p "새 문자열을 프랑스어로 번역하고 PR 올려줘"

🎬 실전 시나리오

🏢시나리오 1: 신입이 프로젝트 파악

상황: 새 프로젝트에 투입됨. 코드 구조를 빨리 이해해야 함

🐛시나리오 2: 긴급 버그 수정

상황: 프로덕션에서 에러 발생!

👥시나리오 3: 팀 코드 리뷰 자동화

상황: 매 PR마다 코드 리뷰 필요

# .claude/agents/reviewer.md 만들고 # GitHub Actions에 연결하면 # PR 올릴 때마다 자동 리뷰! # 또는 수동으로: /review-pr → "보안 이슈 2개, 성능 개선점 1개 발견"
